# ✦ 星球殖民 · 回合制策略游戏 ✦

一个纯前端实现的回合制策略游戏原型，采用模块化架构，无需任何第三方库。

## 📁 项目文件结构

```
p17/
├── index.html              # 游戏入口 HTML 文件
├── css/
│   └── style.css           # 所有样式定义（深空蓝黑 + 荧光绿/橙黄主题）
├── js/
│   ├── main.js             # 主入口，组装所有模块并启动游戏
│   ├── gameState.js        # 游戏状态管理模块（核心）
│   ├── eventSystem.js      # 事件系统（发布-订阅模式）
│   ├── ai.js               # AI 决策模块（简单/困难双难度）
│   └── ui.js               # UI 渲染与交互模块
└── README.md               # 本文档
```

## 🎯 各模块职责

### 1. [gameState.js](file:///d:/zqzl/p17/js/gameState.js) — 游戏状态管理
- **职责**：管理游戏的所有内部状态，确保数据一致性，对外暴露只读接口
- **核心特性**：
  - `getSnapshot()`：深拷贝完整状态快照（支持未来悔棋功能）
  - `saveToHistory()` / `undo()`：历史栈与撤销接口
  - `subscribe()`：观察者模式，状态变更自动通知
  - 所有状态修改必须通过公共方法（`moveUnit` / `developTile` / `buildUnit` / `endTurn`）进行
- **数据结构**：
  - 6x6 二维格子数组，每格记录 `owner` / `output` / `unit`
  - 单位对象分 `player` / `ai` 两组存储，记录 `id` / `x` / `y` / `hasActed`

### 2. [eventSystem.js](file:///d:/zqzl/p17/js/eventSystem.js) — 事件系统
- **职责**：实现模块间解耦通信
- **核心**：`EventEmitter` 类 + `GameEvents` 事件名常量 + `GameEventBus` 全局单例
- **常用事件**：
  - `LOG_MESSAGE`：日志消息
  - `TURN_STARTED` / `TURN_ENDED`：回合流转
  - `UNIT_MOVED` / `UNIT_DEVELOPED` / `UNIT_BUILT`：玩家行动
  - `VICTORY`：胜利判定
  - `DIFFICULTY_CHANGED`：难度切换

### 3. [ai.js](file:///d:/zqzl/p17/js/ai.js) — AI 决策模块
- **职责**：控制 AI 方自动执行最优行动
- **难度**：
  - **简单模式**：30% 概率完全随机行动，70% 最优决策
  - **困难模式**：100% 最优决策
- **决策流程**（详见下方 AI 决策逻辑）

### 4. [ui.js](file:///d:/zqzl/p17/js/ui.js) — UI 渲染与交互
- **职责**：将游戏状态渲染为 DOM，处理所有用户输入
- **包含功能**：
  - 地图渲染：格子归属、单位、产出条、选中/高亮/动画
  - 操作按钮：开发、建造、结束回合
  - 游戏日志：自动滚动到底部，按行动类型着色
  - 弹窗系统：胜利弹窗、重置确认
  - 悬停提示：显示格子详情
  - 键盘快捷键：空格、R、方向键/WASD

### 5. [main.js](file:///d:/zqzl/p17/js/main.js) — 主入口
- **职责**：实例化所有模块，订阅关键事件，触发游戏初始化

## 🎮 游戏规则

### 基本设定
- **地图**：6 × 6 网格
- **双方**：玩家（🚀 蓝色） vs AI（👾 橙色）
- **先手**：玩家
- **单位上限**：每方最多 3 个殖民船
- **产出上限**：每格最多 3 点产出

### 可执行操作（每单位每回合 1 次行动）
| 操作 | 条件 | 效果 |
|------|------|------|
| **移动** | 选中己方单位，点击相邻空格 | 移动单位并占领目标格子 |
| **开发** | 单位站在己方格子上，且产出 < 3 | 该格子产出 +1 |
| **建造** | 单位站在己方格子上，且产出 ≥ 2，且己方单位 < 3 | 消耗 2 点产出，在当前格生成 1 个新单位（本回合不可行动） |
| **结束回合** | 空格键或按钮 | 进入对方回合 |

### 胜利条件
1. **消灭胜利**：对方单位全部被消灭
2. **殖民胜利**：己方单位数 ≥ 3 **且** 己方控制格子数 ≥ 4

### 消灭规则
- 回合切换时，若某单位处于**敌方占领的格子上**，且该格子**至少有一个相邻格子也被敌方占领**，则该单位被消灭

## 🧠 AI 决策逻辑步骤

AI 每回合会循环执行以下流程，直到所有单位都已行动或无合法操作：

### Step 1：难度判断
- **简单模式**：先掷骰子，若 `< 0.3` 则跳到 **Step 2（随机行动）**，否则进入 **Step 3（最优决策）**
- **困难模式**：直接进入 **Step 3**

### Step 2：随机行动（仅简单模式）
1. 从所有未行动的 AI 单位中随机挑一个
2. 收集该单位所有合法操作类型（开发 / 建造 / 移动）
3. 随机选一个操作类型并执行
4. 若无合法操作，返回 false 结束本回合

### Step 3：最优决策（评分排序）
#### 3.1 枚举所有可能操作
对每一个未行动的 AI 单位，枚举其所有合法操作：
- **开发**：若 `canDevelop(unit)` → 生成 1 条候选
- **建造**：若 `canBuild(unit)` → 生成 1 条候选
- **移动**：对 `getMovableTiles(unit)` 中每一格 → 各生成 1 条候选

#### 3.2 对每条候选打分

**开发评分规则**（`_scoreDevelop`）：
```
基础分 = 10 × 产出增量(+1)
加分项：
  + 当前产出为0 → +5    （荒地优先开发）
  + 当前产出为1 → +3
  + 当前产出为2 → +1
  + 开发后产出 ≥ 建造门槛(2) → +4  （为建造铺路）
```

**建造评分规则**（`_scoreBuild`）：
```
基础分 = 18
加分项：
  + 己方单位=1 → +8   （优先扩军）
  + 己方单位=2 → +4
  + 己方格子≥3 → +3   （已有地盘再扩军）
  + 己方单位≥2 且 格子≥3 → +5  （接近胜利条件再拼一把）
减分项：
  - 产出已经=3 → -2   （满级格子用来建造有点亏）
```

**移动评分规则**（`_scoreMove`）：
```
基础分 = 2
加分项：
  + 目标为无主格子 → +5        （圈地优先）
  + 目标为敌方格子 → +8        （占领敌方领地非常优先）
  + (10 - 到最近玩家单位的曼哈顿距离) → 最多+10  （向玩家推进）
  + 若比当前更靠近玩家 → +3
  + 高产出相邻格子数 × 2       （向经济区靠拢）
  + 己方格子<4 → +4           （地盘不足时优先扩张）
```

#### 3.3 选择最高分行动
- 按分数降序排序
- 找出所有最高分的候选（可能并列）
- 从中随机选一个执行（避免 AI 行为过于刻板）

### Step 4：行动间延迟
- 每执行一次行动后 `sleep(500~800ms)`，模拟思考过程，给玩家反应时间
- 回合开始前额外 `sleep(600~1000ms)`

## ⌨️ 键盘快捷键

| 按键 | 功能 | 条件 |
|------|------|------|
| **空格** | 结束当前回合 | 玩家行动阶段 |
| **R** | 重置游戏（二次确认） | 任意时候 |
| **↑/W** | 选中单位向上移动一格 | 玩家行动阶段且已选单位 |
| **↓/S** | 选中单位向下移动一格 | 同上 |
| **←/A** | 选中单位向左移动一格 | 同上 |
| **→/D** | 选中单位向右移动一格 | 同上 |

## 🚀 运行方式

**无需构建，无需安装任何依赖！**

1. 直接用浏览器打开 `index.html` 文件即可
2. 或在项目根目录启动任意静态服务器，例如：
   ```bash
   # Python 3
   python -m http.server 8080
   
   # Node.js (需全局安装 serve)
   serve .
   ```
   然后访问 `http://localhost:8080`

> 💡 **注意**：由于使用了 ES Modules（`type="module"`），建议通过 HTTP 服务器访问，直接双击 `file://` 协议打开在某些浏览器下会被 CORS 策略阻止（Chrome 通常无此问题）。

## 🔮 可扩展方向

1. **悔棋功能**：`GameState` 已内置 `history` 栈和 `undo()` 方法，只需在 UI 加按钮即可
2. **地形系统**：在格子对象中增加 `terrain` 字段，影响移动/产出
3. **特殊单位**：扩展单位类型，如侦查机、战舰等
4. **随机事件**：每回合开始概率触发（陨石、移民潮等）
5. **保存/加载**：`getSnapshot()` 返回的 JSON 直接存 `localStorage`
6. **网络对战**：`GameState` 提供的原子操作非常适合封装为远程 API
